# Claude托管智能体：刚刚发布的重要进展

如果你正在基于Claude进行开发，Claude托管智能体的关键意义并非"又一项智能体功能"，而在于Anthropic向技术栈的上层迈进了一步。

Anthropic不再仅仅提供模型访问和工具原语，而是开始销售长期运行智能体的托管运行时环境：包括智能体定义、云环境、会话管理、事件流、内置工具，以及维持整个系统运行的操作框架。

这将开发者的讨论焦点从：
- "如何搭建智能体循环？"
- "如何实现可恢复性、可观测性和安全性？"

转变为：
- "我的智能体究竟该做什么？"
- "哪些环节我需要控制权，哪些环节我愿意交给Anthropic管理基础设施？"

## 简明答案

截至2026年4月8日，Anthropic已在Claude平台上公开发布Claude托管智能体的测试版。

根据官方文档和发布说明，该产品为开发者提供：
- 预构建的智能体框架
- 托管的云环境
- 长期运行的会话
- 事件持久化和SSE流式传输
- 内置工具（bash、文件操作、网络搜索、网页抓取）
- MCP连接支持

从我们的视角看，这最好理解为Anthropic的托管智能体运行时，而非神奇的"单提示词智能体构建器"。

它直接面向那些希望部署生产级智能体，但不愿自建编排层、沙箱生命周期管理、状态存储、权限系统和可观测性堆栈的团队。

## Claude托管智能体的实质

根据Anthropic文档，托管智能体围绕四个核心概念构建：
- **智能体**：模型、系统提示词、工具、MCP服务器和技能
- **环境**：配置的容器模板
- **会话**：运行的智能体实例
- **事件**：与运行中智能体交换的消息和更新

这一模型比发布口号更重要。

这意味着Anthropic希望开发者按以下结构工作：
1. 一次性定义智能体
2. 一次性定义环境
3. 基于该配置启动会话
4. 通过发送和接收事件来引导执行

这与标准的Messages API请求循环是完全不同的抽象层次。

## 首版发布内容

根据官方文档和公告，公开测试版包含：
- 托管的云容器
- 持久化会话和事件历史
- 框架内置的提示词缓存和压缩
- 容器内的bash访问权限
- 文件读、写、编辑、通配符搜索和grep操作
- 网络搜索和抓取功能
- MCP服务器连接
- 实时会话输出的SSE流式传输
- 会话中断和引导控制

Anthropic同时说明某些功能目前仅处于研究预览阶段：
- 结果管理
- 多智能体协作
- 记忆系统

这一区分很重要。产品虽已公开测试，但并非所有宣传功能都已全面开放。

## 为何比普通API功能更重要

此次发布至少有三个重要意义：

**1. Anthropic正在将框架产品化**
许多智能体产品实质只是：
- 模型调用
- 工具封装
- 重试逻辑
- 伪装成记忆的数据库表

Anthropic明确将框架本身作为产品界面进行封装。

这是战略性的转变。

如果你相信模型行为会持续演进，那么托管框架可能比自定义框架更有价值，因为Anthropic可以随着Claude的发展不断优化调整。

**2. Anthropic将长期运行智能体提升为一等公民工作负载**
文档将托管智能体定位用于：
- 运行数分钟或数小时的任务
- 异步执行
- 有状态会话
- 需要持久化文件系统和服务器端历史记录的工作负载

这不仅仅是"带工具的聊天"。

这是Anthropic在宣告：长期自主工作负载值得拥有自己的运行时模型。

**3. 公司正接近操作系统级抽象**
工程博客比发布公告更清晰地阐明了这一点。

Anthropic正试图稳定智能体运行时的接口，就像操作系统稳定进程和文件的抽象一样。

实践中这意味着：
- 会话成为持久化的真相来源
- 沙箱成为可互换的执行目标
- 框架可随模型改进而变更

这比将假设硬编码到每个应用特定循环中要坚实得多。

## 开发者应首先关注的要点

如果你正在评估此次发布，以下细节至关重要：

**定价策略**
Anthropic表示托管智能体采用标准Claude平台令牌定价，外加每活跃会话小时0.08美元。

定价结构清晰，但也意味着你需要为两个成本维度做预算：
- 模型令牌消耗
- 运行时持续时间

如果你的智能体对话频繁、工具密集或设计为长时间运行，成本建模必不可少。

**访问权限和测试状态**
截至2026年4月8日：
- 产品处于公开测试阶段
- 端点需要`managed-agents-2026-04-01`标头
- API账户默认启用访问权限
- 部分高级功能仍处于研究预览阶段

是的，产品现在真实可用；不，它还不是完全稳定的正式版。

**工具模型**
快速入门指南显示Anthropic推行一种强约定的模型：
1. 创建智能体
2. 创建环境
3. 创建会话
4. 发送用户事件
5. 流式接收响应

这种设计简洁明了，但也意味着你不仅在使用Anthropic的模型，还在采用其控制平面。
