# 用 GPT-5.4 设计令人愉悦的前端界面

来源：https://developers.openai.com/blog/designing-delightful-frontends-with-gpt-5-4

---

GPT-5.4 比其前代版本更擅长网页开发——能生成更具视觉吸引力且更富野心的前端界面。值得注意的是，我们在训练 GPT-5.4 时特别注重提升其 UI 能力和图像运用技巧。通过恰当的引导，该模型能够产出可直接投入生产的前端作品，其中包含精妙的细节处理、精心设计的交互效果以及优美的图像素材。

网页设计可能产生大量不同的结果。优秀的设计需要在克制与创新之间取得平衡——既借鉴经得起时间考验的设计模式，又引入新颖的元素。GPT-5.4 已学习了广泛的设计方法，并理解构建网站的多种不同方式。

当提示信息不够具体时，模型往往会退而采用训练数据中的高频模式。其中一些是经过验证的惯例，但许多只是我们想要避免的过度代表的习惯。结果通常是合理且可用的，但可能趋于通用结构、视觉层次薄弱，以及设计选择未能达到我们脑海中的构想。

本指南将介绍引导 GPT-5.4 实现您所设想设计的实用技巧。

您的浏览器不支持视频标签。

## 模型改进

虽然 GPT-5.4 在[多个维度](https://openai.com/index/introducing-gpt-5-4/)上都有所提升，但对于前端工作，我们重点关注以下三个实际改进：

  * 在整个设计过程中更强的图像理解能力
  * 功能更完整的应用程序和网站
  * 更善于使用工具来检查、测试和验证自身工作

### 图像理解与工具使用

GPT-5.4 经过训练，能够原生使用图像搜索和图像生成工具，使其能够将视觉推理直接融入设计过程。为获得最佳效果，建议指导模型先生成情绪板或多种视觉选项，然后再选择最终素材。

您可以通过明确描述图像应捕捉的属性（例如风格、配色方案、构图或氛围）来引导模型参考强大的视觉素材。还应包含提示指令，指导模型重用先前生成的图像、调用图像生成工具创建新视觉内容，或在需要时引用特定的外部图像。

    默认使用任何已上传/预生成的图像。否则使用图像生成工具创建视觉上令人惊叹的图像素材。除非用户明确要求，否则不要引用或链接网络图像。

### 功能改进

该模型经过训练，能够开发更完整、功能更健全的应用程序。预计模型在长期任务中会更加可靠。您以前认为不可能实现的游戏和复杂用户体验，现在只需一两次交互即可成为现实。

### 计算机使用与验证

GPT-5.4 是我们首个为计算机使用而训练的主流模型。它能够原生导航界面，结合 Playwright 等工具，可以迭代检查其工作、验证行为并优化实现——从而实现更长、更自主的开发工作流程。

观看我们的[发布视频](https://openai.com/index/introducing-gpt-5-4/?video=1170427106%20)，了解这些功能的实际应用。

Playwright 在前端开发中尤其具有价值。它允许模型检查渲染后的页面、测试多种视口、导航应用流程，并检测状态或导航问题。提供 Playwright 工具或技能能显著提高 GPT-5.4 产出精美、功能完整界面的可能性。随着图像理解能力的提升，它还为模型提供了一种视觉验证其工作成果的方式，并能检查其是否与提供的参考 UI 相匹配。

## 实用技巧快速入门

如果你只打算从本文档中采纳少数实践，请从以下几点开始：

  1. 首先选择较低的推理级别。
  2. 预先定义你的设计系统和约束条件（例如，排版、色彩搭配、布局）。
  3. 提供视觉参考或情绪板（例如，附上截图），为模型提供视觉引导。
  4. 预先定义叙述或内容策略，以指导模型的内容创作。

以下是一个入门提示。

    ## 前端任务

    在执行前端设计任务时，避免使用通用、过度复杂的布局。

    **遵循以下硬性规则：**
    - 单一构图：首屏视口必须呈现为一个整体构图，而非仪表盘（除非它确实是仪表盘）。
    - 品牌优先：在品牌页面上，品牌或产品名称必须是核心级别的信号，而不仅仅是导航文本或副标题。任何标题都不应压倒品牌。
    - 品牌测试：如果移除导航后，首屏视口可能属于另一个品牌，则品牌标识过于薄弱。
    - 排版：使用富有表现力、有目的的字体，避免默认字体栈（Inter、Roboto、Arial、系统字体）。
    - 背景：不要依赖单一颜色的平面背景；使用渐变、图像或微妙的图案来营造氛围。
    - 仅限全出血英雄图：在落地页和推广页面上，英雄图像默认应占据主导地位，为边缘到边缘的视觉平面或背景。除非现有设计系统明确要求，否则不要使用内嵌英雄图、侧边栏英雄图、圆角媒体卡片、平铺拼贴或浮动图像块。
    - 英雄区内容限制：首屏视口通常应仅包含品牌标识、一个标题、一句简短的支持性描述、一个行动号召组和一个主导图像。不要在首屏视口中放置统计数据、日程安排、活动列表、地址块、促销信息、“本周”提示、元数据行或次要营销内容。
    - 无英雄区覆盖物：不要在英雄媒体上放置独立的标签、浮动徽章、促销贴纸、信息芯片或提示框。
    - 卡片：默认不使用卡片。英雄区绝不允许使用卡片。卡片仅允许作为用户交互的容器时使用。如果移除边框、阴影、背景或圆角不会损害交互或理解，则不应使用卡片。
    - 每部分一个任务：每个部分应有一个目的、一个标题，通常还有一句简短的支持性描述。
    - 真实的视觉锚点：图像应展示产品、地点、氛围或上下文。装饰性渐变和抽象背景不能作为主要的视觉创意。
    - 减少杂乱：避免使用药丸状集群、统计条、图标行、盒装促销、日程片段和多个相互竞争的文本块。
    - 使用动效来营造存在感和层次感，而非噪音。对于视觉主导的工作，至少应包含 2-3 个有意的动效。

- 色彩与视觉风格：选择清晰的视觉方向；定义CSS变量；避免默认的紫白配色。不偏向紫色系或深色模式。
    - 确保页面在桌面端和移动端均能正常加载。
    - 对于React代码，若团队已采用现代模式，优先使用useEffectEvent、startTransition和useDeferredValue等适用方案。除非代码库已使用，否则不要默认添加useMemo/useCallback；遵循代码库的React编译器指导原则。

    例外情况：若在现有网站或设计系统内工作，需保留既定的模式、结构和视觉语言。

## 提升设计质量的技巧

### 从设计原则出发

定义约束条件，例如：仅使用一个H1标题、不超过六个版块、最多两种字体、一种强调色，以及首屏区域设置一个主要行动号召按钮。

### 构建叙事性页面结构

典型营销页面结构：

  1. 首屏英雄区——确立品牌形象与价值承诺
  2. 辅助视觉区——展示使用场景或环境
  3. 产品详情区——阐释产品特性
  4. 社会证明区——建立可信度
  5. 最终行动号召区——将兴趣转化为行动

### 遵循设计系统规范

鼓励模型在构建初期建立清晰的设计系统。定义核心设计要素，如`背景色`、`表面色`、`主文本色`、`弱化文本色`和`强调色`，以及字体角色如`展示字体`、`标题字体`、`正文字体`和`说明字体`。这种结构有助于模型在整个应用中产出统一且可扩展的UI模式。

对于大多数网页项目，从熟悉的**React与Tailwind**技术栈入手效果显著。GPT-5.4尤其擅长使用这些工具，能更快速地迭代并实现精致成果。

动效与分层UI元素可能增加复杂度，特别是当固定或浮动组件与主要内容产生交互时。处理动画、遮罩层或装饰性图层时，可加入引导说明以确保布局安全性。例如：

    确保固定或浮动UI元素在不同屏幕尺寸下不会遮挡文本、按钮或其他关键内容。将其置于安全区域，必要时置于主要内容后方，并保持充足间距。

### 合理控制推理深度

对于简单网站，过度推理未必更好。实践中，**低至中等推理层级往往能产生更优质的前端成果**，有助于模型保持快速、专注的状态，避免过度设计，同时为复杂设计保留提升推理层级的空间。

### 基于真实内容进行设计

为模型提供真实文案、产品背景或明确项目目标，是提升前端效果最直接的方式之一。这些背景信息能帮助其选择恰当的网站结构，构建更清晰的版块叙事逻辑，并撰写更具说服力的文案，而非依赖通用占位模式。

    # 前端技能

    当作品质量取决于艺术指导、层级、克制、图像和动效而非组件数量时，请使用此技能。

    目标：交付感觉精心设计、高端且符合潮流趋势的界面。默认采用获奖级别的构图：一个核心创意、强烈的图像、简洁的文案、严谨的间距和少量令人印象深刻的动效。

    ## 工作模式

    在构建之前，先写下三件事：

    - 视觉主题：一句话描述情绪基调、材质质感和能量感
    - 内容规划：核心区、支撑区、细节区、最终行动号召
    - 交互主题：2-3个能改变页面感受的动效创意

    每个部分承担一项任务，拥有一个主导的视觉创意，并传达一个主要的要点或行动。

    ## 优雅的默认规则

    - 从构图开始，而非组件。
    - 优先采用全出血式核心区或全画布视觉锚点。
    - 让品牌或产品名称成为最突出的文本。
    - 保持文案足够简短，可在数秒内扫读完毕。
    - 在添加装饰元素前，先运用留白、对齐、比例、裁剪和对比。
    - 限制系统：最多两种字体，默认一种强调色。
    - 默认采用无卡片布局。使用区块、分栏、分隔线、列表和媒体块替代。
    - 将首屏视作海报，而非文档。

    ## 着陆页

    默认顺序：

    1. 核心区：品牌或产品、承诺、行动号召和一个主导视觉
    2. 支撑区：一个具体功能、优惠或证明点
    3. 细节区：氛围、工作流程、产品深度或故事
    4. 最终行动号召：转化、开始、访问或联系

    核心区规则：

    - 仅采用一种构图。
    - 全出血图像或主导视觉平面。
    - 经典全出血规则：在品牌着陆页上，核心区本身必须边缘到边缘，无继承的页面边距、框架容器或共享最大宽度；仅约束内部文本/行动列。
    - 品牌第一，标题第二，正文第三，行动号召第四。
    - 默认避免核心区卡片、数据条、标志云、标签堆砌或悬浮仪表板。
    - 标题在桌面端保持约2-3行，在移动端一眼可读。
    - 保持文本列窄小并锚定在图像的平静区域。
    - 图像上的所有文本必须保持强烈对比和清晰的点击目标。

    如果移除图像后首屏仍能成立，则图像太弱。如果隐藏导航后品牌消失，则层级太弱。

    视口预算：

    - 如果首屏包含固定/粘性页眉，该页眉计入核心区预算。页眉与核心区内容的总和必须在常见桌面和移动尺寸下适配初始视口。

- 使用 `100vh`/`100svh` 全屏区域时，需减去固定界面元素的高度（如 `calc(100svh - 顶部栏高度)`），或将顶部栏设为浮动层而非常规流布局。

    ## 应用界面

    默认采用 Linear 风格的设计约束：

    - 清晰平静的视觉层级
    - 有力的排版与间距控制
    - 少量色彩运用
    - 信息密集但保持可读性
    - 极简的装饰性元素
    - 仅在卡片作为交互载体时才使用卡片样式

    应用界面应围绕以下模块组织：

    - 核心工作区
    - 导航系统
    - 辅助信息面板或检查器
    - 使用单一明确的强调色标识操作或状态

    需避免：

    - 仪表板式的卡片马赛克布局
    - 每个区域都添加粗边框
    - 常规产品界面使用装饰性渐变背景
    - 多种冲突的强调色同时出现
    - 无助于信息扫描的装饰性图标

    若某个面板在取消卡片样式后仍能保持清晰含义，则应移除卡片装饰。

    ## 视觉图像

    图像必须承担叙事功能：

    - 品牌、场馆、内容页及生活方式类产品至少使用一张真实感强的优质图片
    - 优先选用实景摄影而非抽象渐变或虚拟3D物体
    - 选择或裁剪具有稳定色调区域的图片以便文字呈现
    - 避免使用包含嵌入式标识、Logo或与界面冲突的文字杂乱的图片
    - 不使用自带UI框架、分割线、卡片或面板的生成图像
    - 如需呈现多个场景，使用多张独立图片而非合成拼贴

    首屏需要真实的视觉锚点，仅靠装饰性纹理是不够的。

    ## 文案内容

    - 使用产品语言而非设计评论式表达
    - 让标题承担核心表意功能
    - 辅助文案通常只需一个简短句子
    - 删除跨章节的重复内容
    - 不在界面中包含提示性语言或设计评论
    - 每个章节只承担一种职责：解释、验证、深化或转化

    若删除30%的文案能提升页面效果，请继续删减。

    ## 产品界面的功能型文案

    当涉及仪表板、应用界面、管理工具或操作工作区时，默认采用功能型文案而非营销文案：

    - 优先考虑定位、状态和操作说明，而非承诺、情绪或品牌语调
    - 从工作界面本身开始：关键指标、图表、筛选器、表格、状态或任务上下文。除非用户明确要求，否则不添加首屏展示区
    - 章节标题应说明该区域的功能或用户可执行的操作
    - 优秀示例："选定关键指标"、"方案状态"、"搜索指标"、"头部细分"、"最近同步"
    - 避免在产品界面使用理想化首屏标语、隐喻、 campaign 式语言和高层摘要横幅（除非特别要求）
    - 辅助文字应在一句话内说明范围、行为、时效性或决策价值
    - 若某句话可能出现在首页首屏或广告中，应重写直至符合产品界面语感
    - 若某个章节无助于操作、监控或决策，则应移除
    - 核心检验标准：如果操作者仅扫描标题、标签和数字，能否立即理解页面内容？

    ## 动效设计

    用动效建立存在感和层级关系，而非制造噪音。

    视觉主导的作品至少应包含2-3种精心设计的动效：

    - 首屏区域的入场动画序列
    - 一种与滚动关联、粘滞或景深效果
    - 一种强化功能可见性的悬停、展现或布局过渡效果

    优先选用 Framer Motion 实现：

    - 章节展现动画

- 共享布局过渡效果
    - 滚动关联的透明度、位移或缩放变化
    - 粘性叙事效果
    - 推动叙事而非仅填充空间的轮播组件
    - 菜单、抽屉和模态框的呈现动效

    动效原则：

    - 在快速录屏中清晰可辨
    - 移动端表现流畅
    - 快速且克制
    - 全页面保持统一
    - 纯装饰性动效需移除

    ## 硬性规则

    - 默认不使用卡片式设计
    - 默认不采用英雄卡片
    - 需求明确要求全屏展示时，避免使用盒装或居中栏英雄区域
    - 每个区块不超过一个核心概念
    - 区块不应依赖大量微型UI控件进行解释
    - 品牌页面中标题不得压倒品牌标识
    - 杜绝填充性文案
    - 避免分屏式英雄区域，除非文字位于统一平静的视觉侧
    - 无明确理由时字体不超过两种
    - 若产品未建立成熟色彩体系，强调色不超过一种

    ## 需规避的典型问题

    - 首屏即呈现通用SaaS卡片网格
    - 精美图像但品牌存在感薄弱
    - 强势标题却缺乏明确行动指引
    - 文字后方堆砌杂乱图像
    - 重复相同情绪表达的区块
    - 无叙事目的的轮播组件
    - 用堆叠卡片代替布局结构的应用界面

    ## 核心检验标准

    - 首屏能否清晰呈现品牌或产品特征？
    - 是否存在强有力的视觉锚点？
    - 仅浏览标题能否理解页面内容？
    - 每个区块是否职责明确？
    - 卡片设计是否确有必要？
    - 动效是否提升了层级关系或氛围营造？
    - 移除所有装饰性阴影后设计是否仍具高级感？
