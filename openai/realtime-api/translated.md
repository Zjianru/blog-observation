我们近期宣布了最新的语音转语音模型 gpt-realtime，同时正式推出 Realtime API 通用版本及新增 API 功能。Realtime API 与语音转语音模型现已全面开放，在模型质量、可靠性和开发者体验方面实现重大提升。

**模型优化建议**

使用本模型时请注意以下关键建议：

- 在实时交互平台中尝试不同的提示词设计
- 选用 marin 或 cedar 音色可获得最佳助手语音质量
- 为新模型重新设计提示词。由于指令跟随能力的增强，具体指令现在能发挥更强效果
- 注意您提供的具体指令。系统将严格执行所有指令

**API 架构更新**

随着通用版本的发布，我们更新了 Realtime API 架构：现提供测试版接口与通用版接口。建议客户端迁移至通用版接口以获取新功能，测试版接口最终将停止维护。

**功能支持情况**

Realtime API 通用版本包含多项新功能，部分功能在旧版模型中可能无法使用：

| 功能特性 | 通用版模型 | 测试版模型 |
|---------|-----------|-----------|
| 图像输入 | 支持 | 不支持 |
| 长上下文 | 支持 | 支持 |
| 异步函数调用 | 支持 | 不支持 |
| MCP 协议 | 支持（配合异步函数调用效果最佳） | 支持（无异步函数调用时功能受限） |
| 音频转文本 | 支持 | 不支持 |
| 欧盟数据驻留 | 支持 | 仅限 06-03 版本 |

**温度参数调整**

通用版接口已移除温度作为模型参数。测试版接口将温度限制在 0.6-1.2 范围，默认值为 0.8。该模型架构中温度参数作用机制不同，建议用户采用推荐的 0.8 默认值以获得最佳效果。

**新增功能**

**会话闲置超时**：某些应用场景中，用户长时间无输入可能不符合预期。我们新增了自动触发机制，当检测到闲置时会提示："您还在吗？" 可通过在 session_create 中设置 idle_timeout_ms 参数启用此功能。
