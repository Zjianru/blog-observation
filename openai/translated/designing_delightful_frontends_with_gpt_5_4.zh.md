# 使用 GPT-5.4 设计令人愉悦的前端

![](../images/designing-delightful-frontends-with-gpt-5-4_00.png)

作者：Brian Fioca、Alistair Gillespie、Kevin Leneway、Robert Tinn

GPT-5.4 是一个比其前辈更好的 Web 开发者——能生成更具视觉吸引力和更大胆的前端。值得注意的是，我们在训练 GPT-5.4 时重点关注了改进的 UI 能力和图像使用。有了正确的引导，模型可以生成包含微妙细节、精良交互和精美图像的生产级前端。

网页设计可以产生大量的结果空间。优秀的设计在克制与创新之间取得平衡——从经得起时间考验的模式中汲取营养，同时引入新的元素。GPT-5.4 已经学会了这种广泛的设计方法谱系，并理解构建网站的不同方式。

当提示词不够具体时，模型往往会退回到训练数据中的高频模式。其中一些是经过验证的约定，但许多只是我们想要避免的过度代表的习惯。结果通常是可信且可用的，但可能会偏离通用的结构、弱的视觉层次，以及与我们脑海中设想的不相符的设计选择。

本指南阐述了在引导 GPT-5.4 按你设想的方式设计方面的实用技术。

## 模型改进

虽然 GPT-5.4 在多个方面都有改进，但就前端工作而言，我们专注于三个实际收益：

- 在整个设计过程中更强的图像理解能力
- 更功能完整、应用和网站
- 更好地使用工具来检查、测试和验证自己的作品

### 图像理解和工具使用

GPT-5.4 被训练为原生使用图像搜索和图像生成工具，使其能够将视觉推理直接融入设计过程。为了获得最佳效果，可以指示模型先生成一个情绪板或几个视觉选项，然后再选择最终素材。

你可以通过明确描述图像应该捕获的属性（如风格、色彩调色板、构图或情绪）来引导模型获得强大的视觉参考。你还应该包含提示指令，引导模型重用先前生成的图像、调用图像生成工具创建新的视觉效果，或在需要时引用特定的外部图像。

```
默认使用任何已上传/预生成的图像。否则使用图像生成工具创建视觉上令人惊叹的图像成品。除非用户明确要求，否则不要引用或链接到网络图像。
```

### 功能改进

模型被训练为开发更完整且功能可靠的应用程序。预计模型在长时程任务中会更可靠。你之前认为不可能的游戏和复杂用户体验在一到两轮内就能实现。

### 计算机使用与验证

GPT-5.4 是我们的首款为主流模型训练的计算机使用能力。它能本机导航界面，结合 Playwright 等工具，可以迭代检查自己的工作、验证行为并完善实现——实现更长的、更自主的开发工作流程。

### 实用技巧速成

如果你只从本文档中采纳少数几个实践，请从以下开始：

- 选择低推理级别开始。
- 提前定义你的设计系统和约束（即字体排版、色彩调色板、布局）。
- 指定具体的参考设计或设计系统。
- 让模型首先创建参考图像和情绪板。
- 给模型时间思考设计选择。
- 指导模型验证自己的工作。

### 设计系统与约束

在开始之前定义约束。明确的约束会引导模型生成更加一致和精确的设计。如果你知道你的项目使用特定的字体、色彩调色板或布局网格，在你的提示中声明它们。GPT-5.4 善于遵循并应用你提供的设计系统。

### 参考设计

提供具体的参考设计或设计系统链接会显著提高输出的相关性。参考不必完全匹配你想要的输出——它们为模型提供了一个视觉起点。参考帮助你避免冗长的文字描述，并更精确地传达风格、情绪和构图。

### 图像引导的设计

GPT-5.4 在图像理解和工具使用方面有显著提升。在设计过程中集成图像生成允许模型创建情绪板、探索视觉选项并选择与你愿景一致的最佳资产。

当你提供参考 UI 时，模型可以使用 Playwright 验证其工作是否与参考匹配。这使得迭代设计改进更加直接。

### 使用 Frontend Skill 整合一切

Frontend Skill 整合了本指南中描述的所有技术。通过使用 Frontend Skill，你可以访问：

- 预配置的 GPT-5.4 提示模板
- Playwright 工具集成
- 图像生成工具集成
- 设计系统默认值

### 关键要点

GPT-5.4 在前端设计方面代表了显著的进步。通过正确的引导，模型可以生成生产就绪的、视觉上令人愉悦的界面。以下是开始的最重要提示：

1. 定义你的设计系统并提前设置约束
2. 提供具体的参考设计或视觉起点
3. 使用图像生成工具创建情绪板和选项
4. 集成 Playwright 以验证和迭代设计
5. 给模型时间和空间来思考设计选择

有了这些技术，你就可以使用 GPT-5.4 来创建真正令人愉悦的前端。
