# 使用 Codex 与 Figma 构建前端用户界面

来源：https://developers.openai.com/blog/building-frontend-uis-with-codex-and-figma

---

从今天起，您可以通过 Figma MCP 服务器，利用 Codex 生成 Figma 设计文件。该 MCP 服务器专为双向协作而构建，既能将可运行的 UI 轻松带入 Figma 画布，也能无缝转换回代码，让您得以在最佳创意（而非仅最初想法）的基础上持续构建。

Codex 桌面应用程序专为智能体编码而生。它提供了一个聚焦的界面，支持跨项目并行管理多个智能体、在保持上下文连贯的同时追踪进度，并能集成外部工具。这种流畅的协作体验令人熟悉。在 Figma 中，团队同样能灵活协作。画布专为探索与迭代设计，是让最佳创意得以成型的空间。通过将 Figma 画布与 Codex 连接，这种探索精神直接延伸至开发工作流——为用户解锁了一种强大的新方式，使其能够构建从原型到生产应用的一切。

### 从设计启动应用

Figma MCP 服务器的核心用例之一，是从 Figma 文件中提取上下文，并将该上下文用于代码生成。该服务器能够从 Figma Design、Make 及 FigJam 文件中捕获信息，并将其作为构建流程的一部分传递给 Codex。

开始前，请打开您计划基于其构建应用的 Figma 文件。右键点击选择画框，点击“复制为”并选择“复制所选内容的链接”。

您的浏览器不支持视频标签。

这些选择链接直接关联到 Figma 画布上的某个画框或节点。它们可能是一个单一元素或一组组件，但本质上是智能体将用于代码生成的源数据。选择可来自 Figma Design、Make 或 FigJam 文件。获取链接后，打开 Codex 并选择新建或现有项目。随后，您可以通过如下指令提示 Codex：

    请帮我用代码实现这个 Figma 设计，尽可能使用我现有设计系统的组件。

您的浏览器不支持视频标签。

类似这样的提示将指示代理调用来自 Figma MCP 服务器的 `get_design_context` 工具。该工具有助于从 Figma 文件中提取关键设计信息，如布局、样式和组件信息，然后将这些上下文提供给 Codex 用于代码生成。

除了提取设计信息外，Figma MCP 服务器还提供了许多其他实用工具；完整列表请务必[查阅我们的文档](https://developers.figma.com/docs/figma-mcp-server/tools-and-prompts/)。

### 从代码到画布

在代码中迭代后，您会希望将设计带回画布，以比较流程、探索替代方案并验证您的假设。Figma MCP 服务器让您能够轻松地将这些屏幕带回 Figma，而无需从浏览器手动重新创建。使用 `generate_figma_design` 工具，您可以在几秒钟内将实时运行的界面转换为完全可编辑的 Figma 框架——将真实、可运行的 UI 直接带入画布，以便进行更深入的探索和协作。

首先，您需要渲染应用程序的 UI。这可以在本地完成，也可以通过公开可用的网络服务器完成。然后，请 Codex 帮助您生成一个新的 Figma 设计文件。

您的浏览器不支持视频标签。

Codex 将引导您完成以下步骤：

  1. 决定是创建新的 Figma 文件还是使用现有文件。
  2. 确定将文件放置在哪个工作区。
  3. 设置应用程序以进行 UI 捕获。
  4. 为您的应用程序打开一个新的浏览器会话。

当应用程序重新加载时，您将在页面顶部看到一个新的工具栏，其中包含以下选项：

**整个屏幕：** 捕获当前显示的整个屏幕的渲染，并将其发送到 Figma 文件。

**选择元素：** 选择页面上特定的组件或元素进行捕获

**打开文件：** 打开 Figma 文件以检查您的新设计图层

您的浏览器不支持视频标签。

在捕获了所有要传输到 Figma 的信息后，您可以选择打开文件或返回 Codex。Codex 将为您准备好 Figma 文件的 URL。

### 一次往返之旅：MCP的故事

现在，您已经构建了应用程序并设置了设计文件，接下来可以开始迭代了。在这里，您可以充分利用Figma画布提供的所有功能，包括：

  * 添加设计系统组件
  * 将样式、字体和颜色更新为变量
  * 调整布局并添加注释说明
  * 设计各种交互和空状态
  * 协作进行多版本和探索性设计

完成UI的优化后，您可以按照本文开头概述的相同步骤，通过MCP服务器将这些更改拉取回您的应用程序中。

当代码与画布相互连接时，您可以在执行与探索之间流畅切换。这种往返流程释放了Figma MCP服务器与Codex结合后的真正力量——能够从任何起点出发，打造有意义、高质量的应用程序体验，同时不牺牲开发速度。

要了解更多关于Figma MCP服务器的信息，请查阅我们的[文档](https://developers.figma.com/docs/figma-mcp-server/)，或直接在Codex桌面应用程序中安装Figma MCP服务器以开始使用。
