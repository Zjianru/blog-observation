更新：我们已将Agent Skills发布为跨平台可移植性的开放标准。（2025年12月18日）

随着模型能力的提高，我们现在可以构建与完整计算环境交互的通用代理。例如，Claude Code可以使用本地代码执行和文件系统跨领域完成复杂任务。但随着这些代理变得越来越强大，我们需要更可组合、可扩展和可移植的方式来为它们配备特定领域的专业知识。

这引导我们创建了Agent Skills：有序的指令、脚本和资源文件夹，代理可以动态发现和加载，以更好地执行特定任务。Skills通过将您的专业知识打包为Claude的可组合资源来扩展Claude的能力，将通用代理转变为适合您需求的专用代理。

为代理构建技能就像为新员工准备入职指南一样。与其为每个用例构建分散的、定制的代理，现在任何人都可以通过捕获和共享他们的程序性知识来使用可组合的能力来专门化他们的代理。在本文中，我们解释什么是Skills，它们如何工作，并分享构建自己Skills的最佳实践。

技能是一个目录，包含SKILL.md文件，其中包含有序的指令、脚本和资源，为代理提供额外的能力。

技能和代码执行

技能还可以包含Claude可酌情执行的代码作为工具。

大型语言模型擅长许多任务，但某些操作更适合传统代码执行。例如，通过token生成对列表进行排序比简单运行排序算法要昂贵得多。除了效率问题外，许多应用程序需要只有代码才能提供的确定性可靠性。

在我们的示例中，PDF技能包括一个预写的Python脚本，读取PDF并提取所有表单字段。Claude可以运行此脚本，而无需将脚本或PDF加载到上下文中。而且因为代码是确定性的，此工作流是一致的和可重复的。

技能还可以包含Claude可酌情执行的代码作为工具，基于任务的性质。

使用技能时的安全注意事项

技能通过指令和代码为Claude提供新能力。虽然这使它们变得强大，但也意味着恶意技能可能在其使用的环境中引入漏洞，或指示Claude泄露数据并采取意想不到的操作。

我们建议仅从可信来源安装技能。在安装来自不太可信来源的技能时，使用前彻底审核。首先阅读捆绑在技能中的文件内容以了解其作用，特别注意代码依赖项和捆绑的资源（如图像或脚本）。同样，注意技能中指示Claude连接到可能不可信的外部网络源的指令或代码。

技能的未来

Agent Skills今天在Claude.ai、Claude Code、Claude Agent SDK和Claude开发者平台上都得到支持。

在接下来的几周内，我们将继续添加支持创建、编辑、发现、共享和使用Skills的完整生命周期的功能。我们特别兴奋地看到技能如何帮助组织和个人与Claude共享他们的上下文和工作流。我们还将探索技能如何通过教代理涉及外部工具和软件的更复杂工作流来补充模型上下文协议（MCP）服务器。

进一步展望，我们希望使代理能够自己创建、编辑和评估技能，让它们将自己的行为模式编纂成可重用的能力。

技能是一个简单概念，具有同样简单的格式。这种简单性使组织、开发者和最终用户能够更轻松地构建定制代理并赋予它们新能力。
