在Claude Code中，Claude与您一起编写、测试和调试代码，导航您的代码库、编辑多个文件并运行命令来验证其工作。给予Claude这么多对您代码库和文件的访问权限会带来风险，特别是在提示注入的情况下。

为了帮助解决这个问题，我们在Claude Code中引入了两个基于沙箱的新功能，两者都旨在为开发者提供更安全的工作场所，同时允许Claude更自主地运行并减少权限提示。在我们的内部使用中，我们发现沙箱将权限提示安全地减少了84%。通过定义Claude可以自由工作的固定边界，它们提高了安全性和代理能力。

在Claude Code上保持用户安全

Claude Code运行在基于权限的模型上：默认情况下，它是只读的，这意味着它在进行任何修改或运行任何命令之前都会请求许可。也有一些例外：我们自动允许安全命令如echo或cat，但大多数操作仍需要明确批准。

不断点击"批准"会减慢开发周期，并可能导致"批准疲劳"，用户可能不会密切关注他们批准的内容，反而使开发变得不那么安全。

为了解决这个问题，我们为Claude Code推出了沙箱功能。

沙箱：一种更安全、更自主的方法

沙箱在被允许的边界内创建预定义的边界，Claude可以更自由地工作，而不是为每个操作请求许可。启用沙箱后，您会得到大大减少的权限提示和更高的安全性。

值得注意是，有效的沙箱需要文件系统和网络隔离。没有网络隔离，受损的代理可以泄露敏感文件如SSH密钥；没有文件系统隔离，受损的代理可以轻松逃逸沙箱并获得网络访问。正是通过使用这两种技术，我们才能为Claude Code用户提供更安全、更快速的代理体验。

Claude Code中两个新的沙箱功能

沙箱化bash工具：无需权限提示的安全bash执行

我们引入了一个新的沙箱运行时，作为研究预览版提供，让您可以精确定义代理可以访问哪些目录和网络主机，而无需启动和管理容器的开销。这可用于沙箱任意进程、代理和MCP服务器。它也可作为开源研究预览版提供。

在Claude Code中，我们使用此运行时来沙箱bash工具，允许Claude在您设定的限制内运行命令。在安全的沙箱内，Claude可以更自主地运行并安全地执行命令，而无需权限提示。如果Claude试图访问沙箱外的任何内容，您会立即收到通知，并可以选择是否允许。

我们建立在Linux bubblewrap和MacOS seatbelt等操作系统级原语之上，以在操作系统级别强制执行这些限制。它们不仅涵盖Claude Code的直接交互，还涵盖命令生成的任何脚本、程序或子进程。如上所述，此沙箱同时强制执行：

两个组件都是可配置的：您可以轻松选择允许或禁止特定文件路径或域。

Claude Code的沙箱架构通过文件系统和网络控制隔离代码执行，自动允许安全操作，阻止恶意操作，仅在需要时请求权限。

沙箱确保即使是一次成功的提示注入也被完全隔离，无法影响整体用户安全。这样，受损的Claude Code无法窃取您的SSH密钥或向攻击者的服务器发送信息。

云端Claude Code：在云端安全运行Claude Code

今天，我们还发布了Claude Code on the Web，使用户能够在云中隔离的沙箱中运行Claude Code。Web上的Claude Code在隔离的沙箱中执行每个Claude Code会话，在安全的方式下完全访问其服务器。我们设计这个沙箱以确保敏感凭证（如git凭证或签名密钥）永远不会与Claude Code一起在沙箱中。这样，即使沙箱中运行的代码受损，用户也能免受进一步伤害。

Web上的Claude Code使用自定义代理服务来透明处理所有git交互。在沙箱内，git客户端使用自定义构建的范围凭证向此服务进行身份验证。代理验证此凭证和git交互的内容（例如，确保它仅推送到配置的分支），然后在将请求发送到GitHub之前附加正确的身份验证令牌。

Claude Code的Git集成通过安全代理路由命令，验证身份验证令牌、分支名称和仓库目的地——允许安全的版本控制工作流，同时防止未经授权的推送。
