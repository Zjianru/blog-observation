扩展思考更新

2025年12月15日

扩展思考能力自发布以来有所改进，我们建议在大多数情况下使用该功能而不是专用思考工具。扩展思考提供类似的好处——为Claude提供空间来推理复杂问题——具有更好的集成和性能。请参阅我们的扩展思考文档以获取实现细节。

随着我们继续增强Claude的复杂问题解决能力，我们发现了一种特别有效的方法：一种"思考"工具，在复杂任务期间为结构化思维创建专用空间。

这个简单而强大的技术——正如我们将在下面解释的，它不同于Claude的新"扩展思考"能力（有关扩展思考实现细节请参阅此处）——已经导致Claude的智能体工具使用能力显著改进。这包括遵循策略、做出一致的决策和处理多步问题，所有这些都具有最小的实现开销。

在这篇文章中，我们将探讨如何在不同的应用程序中实现"思考"工具，基于经过验证的基准测试结果为开发者提供实用指导。

什么是"思考"工具？

使用"思考"工具，我们让Claude有能力在得出最终答案的过程中包含一个额外的思考步骤——带有自己指定的空间。

虽然听起来相似，但这是不同的概念。扩展思考是关于Claude在开始生成响应之前做什么。使用扩展思考，Claude在采取行动之前深入思考和迭代其计划。"思考"工具是用于Claude在开始生成响应后，添加一个步骤来停下来思考它是否有所需的所有信息来继续。这在执行长链工具调用或与用户的长时间多步对话中特别有帮助。

这使得"思考"工具更适合于这样的情况：Claude没有用户查询 alone 所需的所有信息来制定其响应，并且需要处理外部信息（例如工具调用结果中的信息）。使用"思考"工具进行的推理不如扩展思考所能获得的全面，主要集中在模型发现的新信息上。

我们建议将扩展思考用于更简单的工具使用场景，如非顺序工具调用或直接的指令遵循。扩展思考也适用于编码、数学和物理等用例，当您不需要Claude调用工具时。"思考"工具更适合于Claude需要调用复杂工具、在长链工具调用中仔细分析工具输出、导航具有详细指南的策略密集型环境，或做出顺序决策（每一步都建立在前一步的基础上且错误代价高昂）的场景。

以下是一个使用来自τ-Bench的标准工具规范格式的示例实现：

{
  "name": "think",
  "description": "Use the tool to think about something. It will not obtain new information or change the database, but just append the thought to the log. Use it when complex reasoning or some cache memory is needed.",
  "input_schema": {
    "type": "object",
    "properties": {
      "thought": {
        "type": "string",
        "description": "A thought to think about."
      }
    },
    "required": ["thought"]
  }
}

性能测试

我们使用τ-bench（tau-bench）评估了"思考"工具，τ-bench是一个全面的基准测试，旨在测试模型在现实客户服务场景中使用工具的能力，其中"思考"工具是评估标准环境的一部分。

τ-bench评估Claude的能力：

主要评估指标是pass^k，它衡量给定任务的所有k个独立任务试验成功的概率，跨所有任务平均。与其他LLM评估常见的pass@k指标（衡量k次试验中至少一次成功）不同，pass^k评估一致性和可靠性——这对客户服务应用程序至关重要，一致的策略 adherence 是必不可少的。

## 使用思考工具

在采取任何行动或响应用户（在收到工具结果之后）之前，使用思考工具作为草稿纸来：
- 列出适用于当前请求的具体规则
- 检查是否收集了所有必需信息
- 验证计划的操作符合所有策略
- 迭代检查工具结果的正确性

以下是在思考工具内部迭代的示例：
<think_tool_example_1>
用户想要取消航班ABC123
- 需要验证：用户ID、预订ID、原因
- 检查取消规则：
  * 是否在预订后24小时内？
  * 如果不是，检查机票级别和保险
- 验证没有航段已飞行或在过去
- 计划：收集缺失信息、验证规则、获得确认
</think_tool_example_1>

<think_tool_example_2>
用户想要预订3张前往纽约的机票，每人2件托运行李
- 需要用户ID来检查：
  * 会员级别对应的免费行李额
  * 档案中存在哪些支付方式
- 行李计算：
  * 经济舱 × 3位乘客
  * 如果是普通会员：每人1件免费行李 → 3件额外行李 = $150
  * 如果是银卡会员：每人2件免费行李 → 0件额外行李 = $0
  * 如果是金卡会员：每人3件免费行李 → 0件额外行李 = $0
- 需要验证的支付规则：
  * 最多1张旅行证书、1张信用卡、3张礼品卡
  * 所有支付方式必须在档案中
  * 旅行证书的余额将被浪费
- 计划：
1. 获取用户ID
2. 验证会员级别对应的行李费
3. 检查档案中的支付方式及其组合是否允许
4. 计算总计：机票价格 + 任何行李费
5. 获得预订的明确确认

特别有趣的是不同方法的比较。使用"思考"工具加优化提示取得了明显优于扩展思考模式的结果（扩展思考表现类似于无提示的"思考"工具）。单独使用"思考"工具（不带提示）比基线有所改进，但仍不如优化方法。

"思考"工具与优化提示的组合以显著优势取得了最强表现，这可能是因为基准测试的航空策略部分复杂性很高，模型从被给予如何"思考"的示例中获益最大。

在SWE-Bench上的性能

当评估Claude 3.7 Sonnet时，类似的"思考"工具被添加到我们的SWE-bench设置中，为实现的0.623的最先进分数做出了贡献。改编的"思考"工具定义如下：

{
  "name": "think",
  "description": "Use the tool to think about something. It will not obtain new information or make any changes to the repository, but just log the thought. Use it when complex reasoning or brainstorming is needed. For example, if you explore the repo and discover the source of a bug, call this tool to brainstorm several unique ways of fixing the bug, and assess which change(s) are likely to be simplest and most effective. Alternatively, if you receive some test results, call this tool to brainstorm ways to fix the failing tests.",
  "input_schema": {
    "type": "object",
    "properties": {
      "thought": {
        "type": "string",
        "description": "Your thoughts."
      }
    },
    "required": ["thought"]
  }
}

我们的实验（n=30个带"思考"工具的样本，n=144个不带"思考"工具的样本）表明，包含此工具的孤立效应平均提高了1.6%的性能（Welch's t检验：t(38.89)=6.71，p<.001，d=1.47）。

实现最佳实践

为了从Claude的"思考"工具中获得最大收益，我们基于τ-bench实验推荐以下实现实践。

1. 具有特定领域示例的战略提示

2. 将复杂指导放在系统提示中

我们发现，当指导冗长和/或复杂时，将关于"思考"工具的指令放在系统提示中比放在工具描述本身中更有效。这种方法提供了更广泛的上下文，并帮助模型更好地将思维过程整合到其整体行为中。

结论

我们的研究证明，"思考"工具可以显著增强Claude 3.7 Sonnet在需要策略 adherence 和长链工具调用推理的复杂任务上的性能。"思考"不是一刀切的解决方案，但对于正确的用例，它提供了实质性的好处，同时实现复杂性最小。
